# Seeking Clarity in Life:

I am a programmer and will continue to be so for another 20 to 30 years. By the looks of it, I can still work remotely post my retirement (age: 60)

I have the option to choose to be a lousy programmer or a quality programmer. I will define below what constitutes each:

| Lousy Programmer | Best Programmer |
| ---- | ---- |
| Shallow Knowledge | Deep Knowledge |
| Improper testing | Testing is viewed as importantly as developing the code |
| Low quality code | High quality code |
| Shies away from criticism | Faces criticism and accepts them with grace |
| Prioritises comfort zone | Can stray away from Comfort Zone | 

A wise man once told me, programming is easy. Anyone can learn it. Learning the attitude to continuosly improve oneself is the hard part. This growing requires us to tear away our ego and build toward the one thing that matters. The product that you are building.
