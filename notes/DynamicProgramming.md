# A Brief Understanding of the Principles of Dynamic Programming:

## What is it?
Dynamic Programming allows us to design algorithms that systematically search all possibilities while storing results to avoid recomputing.

Consider the case of the finding the nth fibonacci number:

```
0 1 1 2 3 5 8 11 19 30 ...
```

Each fibonacci numbers are generated by adding the previous two numbers in the series. One method to solve is coded in the below:

```
def fibonacci(n):
	if n == 0:
		return 0
	elif n == 1:
		return 1
	return fibonacci(n-1) + fibonacci(n-2)
```

### The problem with the above approach:

1. Since we do not store the intermediate results anywhere, every time the function is called, we end up recalculating results that were generated in a previous run. That's a lot of computation time there.
2. For large numbers, our program takes exponential time

### The Solution:

Cache the intermediate results:
Our program need not recursively call itself until it reaches the base case. If there was a computation that was done for *n - 1* and *n - 2*, we can safely store them in an array. This process of storing intermediate results is referred to as **memoisation**

```
mem = [0,1]
def fibonacci(n):
	while len(men) <= n:
		mem.append(mem[-1] + mem[-2])
	return mem[n]
```

The above function now runs in O(n) time at the cost of O(n) space (for the mem array)
